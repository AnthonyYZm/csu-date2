## 项目概述

请帮我搭建一个校园约会匹配平台的前端，模仿 Stanford DateDrop（https://stanford.trydatedrop.com/）的产品逻辑。

核心理念：**不用刷屏，每周给你匹配一个最合拍的人。**

技术栈：**Next.js 14 (App Router) + TypeScript + Tailwind CSS**
（如果你用其他框架，把这行改成 React / Vue / 纯 HTML 等）

---

## 全局设计风格

- 温暖、柔和、有亲和力，不要冷硬的科技感
- 配色以暖色调为主：奶油白背景 `#FBF8F4`，主强调色珊瑚橙 `#E8A87C`，辅助粉 `#D4A0A0`，辅助紫 `#C2B8D6`
- 中文字体用 `Noto Sans SC`，英文标题可用衬线体（如 `Playfair Display`）做点缀
- 圆角偏大（12-16px），卡片用细边框 `1px solid #E8E6DF`
- 交互要有微动画：按钮 hover 上浮、页面切换淡入淡出、进度条平滑过渡
- 移动端优先，所有页面必须响应式

## 页面结构

整个前端包含以下页面/状态，可以做成单页应用（SPA），也可以多路由：

### 页面 1：首页（Landing Page）

**目标**：30 秒内让用户理解产品并点击注册。

#### 顶部导航
- 左侧：Logo（一个小圆点 + 文字「[你的学校名] date drop」）
- 右侧：「登录」按钮（文字按钮）

#### Hero 区域
- 大标题：「不刷屏，每周遇见一个**对的人**」
  - "对的人" 三个字用衬线体+斜体+强调色，制造视觉焦点
- 副标题：「告诉我们你是谁、你在乎什么。每周二，我们把一个和你最合拍的人送到你的邮箱。」
- CTA 按钮：「开始匹配 →」，珊瑚橙背景、白色文字、大圆角、hover 上浮
- 下方统计数字（三列）：
  - 「X,XXX 已注册」「X,XXX+ 成功配对」「每周二 推送匹配」
  - 数字大号加粗，标签小号灰色

#### How it works 区域
三张卡片横排（移动端竖排）：
1. **01 填问卷** — 花 2 分钟告诉我们你的价值观和偏好
2. **02 每周 Opt-in** — 周二晚 9 点前点「我参加」
3. **03 查看匹配** — 收到邮件，了解为什么你们合适

每张卡片：序号用强调色大字，标题加粗，描述文字灰色。hover 时卡片微微上浮。

#### 底部 Footer
- 链接：关于我们 / 隐私政策 / 联系我们
- 「仅限 @[学校邮箱域名] 邮箱注册」

---

### 页面 2：问卷流程（Questionnaire）

**目标**：2-3 分钟内完成，体验流畅不枯燥。

#### 全局 UI
- 顶部：「← 返回」按钮 + 进度条 + 步骤标签 「1/3」
- 进度条：细长条，已完成部分填充强调色，有平滑动画
- 每一步切换时有淡入淡出过渡（300ms）

---

#### Step 1：基本信息

标题：「先认识一下你」
副标题：「这些信息帮我们找到合适的候选人」

三个问题，每个都是**按钮组单选**（不是下拉菜单）：

**你的性别**
选项：`男` / `女` / `非二元`

**你想找**
选项：`约会对象` / `朋友` / `都行`

**你的年级**
选项：`大一` / `大二` / `大三` / `大四` / `研究生`

按钮组样式：
- 默认：白色背景、灰色细边框、圆角
- 选中：强调色浅底 + 强调色边框 + 文字变色
- 每组按钮横排排列，等宽

底部：「下一步 →」按钮（全部选完后才可点击，未完成时半透明禁用）

---

#### Step 2：核心价值观

标题：「你最在乎什么？」
副标题：「选择 4 个对你最重要的核心价值观」

展示 15 个价值观标签，网格排列（3-4 列）：

```
好奇心  冒险  诚实  独立  善良
忠诚  家庭  野心  创造力  幽默
勇气  公平  自由  自律  信念
```

交互逻辑：
- 每个标签可点击选中/取消
- 最多选 4 个，选满后其他标签变半透明且不可点
- 选中标签：强调色背景 + 白色文字，有个小勾或加粗边框
- 未选中：白色背景、灰色边框
- 标签前可以加一个小符号/icon 增加趣味性

实时显示「已选择 X/4」计数器

底部：「下一步 →」（选满 4 个后可点击）

---

#### Step 3：态度量表

标题：「你怎么看？」
副标题：「拖动滑块到最符合你的位置」

6 道题，每题一个 **1-7 滑块**（slider）：

| 题目文本 | 左端标签 | 右端标签 |
|---------|---------|---------|
| 拥有孩子对我来说很重要 | 完全不同意 | 非常同意 |
| 比起社交，我更喜欢独处充电 | 外向型 | 内向型 |
| 我愿意为事业牺牲个人时间 | 生活优先 | 事业优先 |
| 在恋爱中我希望保持频繁联系 | 给我空间 | 随时联系 |
| 遇到矛盾我倾向于直接沟通 | 回避冲突 | 正面沟通 |
| 我是一个夜猫子 | 早起鸟 | 夜猫子 |

每题布局：
- 题目文本（16px，加粗）
- 滑块（自定义样式：轨道 6px 高、圆角、灰色底；滑块圆形、强调色、带白色边框和阴影）
- 滑块下方两端标签（12px，灰色），中间显示当前数值

底部：「提交问卷 ✓」（全部滑动过后可点击）

---

### 页面 3：完成页

**目标**：确认提交成功，告知下一步。

#### 布局
居中一张卡片：
- 顶部：绿色圆形勾 ✓（带入场动画，从小变大弹出）
- 标题：「你已加入匹配池！」
- 副标题：「我们会在每周二晚上把你的最佳匹配发到你的学校邮箱。」
- 信息摘要（灰底区域）：
  - 身份：男 · 大三 · 寻找约会对象
  - 价值观：好奇心、冒险、诚实、独立
  - 问卷：6 题已完成
- 底部提醒文字：「记得每周二晚 9 点前点「我参加」才会被匹配哦」

## 数据结构参考

问卷收集的数据最终要发送给后端，结构如下：

```typescript
interface QuestionnaireData {
  // Step 1
  gender: '男' | '女' | '非二元';
  seeking: '约会对象' | '朋友' | '都行';
  grade: '大一' | '大二' | '大三' | '大四' | '研究生';

  // Step 2：从 15 个里选 4 个，存 id
  values: string[];  // e.g. ['curiosity', 'adventure', 'honesty', 'independence']

  // Step 3：每题 1-7 整数
  likert: {
    children: number;    // 孩子重要性
    introvert: number;   // 内外向
    career: number;      // 事业vs生活
    contact: number;     // 联系频率
    conflict: number;    // 冲突处理
    sleep: number;       // 作息习惯
  };
}
```

当前阶段前端不需要真的调接口，用本地 state 管理即可。提交后直接跳转完成页。

## 注意事项

1. **所有文案用中文**，界面语言统一中文
2. **不需要后端对接**，数据存在前端 state 里
3. **不需要登录功能**，假设用户已通过邮箱验证
4. **滑块必须自定义样式**，不要用浏览器默认的丑滑块
5. **移动端体验很重要**，问卷在手机上填写的概率很高
6. 按钮组选中态要有明显的视觉反馈
7. 页面切换要有过渡动画，不要生硬跳转
8. 进度条要平滑过渡，不要跳变
