# CSU DateDrop 深度问卷系统 — AI 开发 Prompt

> 参考 SJTU Date 问卷体系（56 题），结合中南大学校园实际，重新设计完整问卷。
> 把这份文档 + code.html 一起粘贴给 AI，让它实现完整的问卷前端。

## 一、问卷设计理念

### 1.1 和现有简版问卷的关系

现有 code.html 中是一个 3 步、14 题的简版问卷（基本信息 + 价值观 + 6 道 Likert）。
**本文档是完整版问卷的设计稿，用于替换简版问卷。**

### 1.2 核心设计原则

- **自我特征 + 伴侣偏好双轨制**：关键维度同时问"你自己是什么样"和"你希望对方是什么样"（参考 SJTU Date 的 a/b 双题结构），这是匹配算法的精度关键
- **「标为重要」加权机制**：每道题下方有一个「☆ 设为重要」按钮，用户点击后该题在匹配算法中权重翻倍。这让用户自己告诉算法"哪些维度对我最重要"
- **题目分模块、带呼吸感**：每个模块有小标题和一句引导语，不是冷冰冰的填表
- **中南特色融入**：校区、食堂、岳麓山、湘江等元素自然嵌入题目中

### 1.3 总题量与预计时长

- 总计 **50 题**（含子题），分 **7 个模块**
- 预计填写时间 **8-12 分钟**
- 每个模块一个页面，顶部有进度条（1/7 → 7/7）

## 二、数据结构

```typescript
interface FullQuestionnaire {
  // 模块一：基本信息（必填，不参与相似度计算，用于过滤）
  gender: '男' | '女' | '非二元';
  sexuality: '异性恋' | '同性恋' | '双性恋' | '其他';
  grade: '大一' | '大二' | '大三' | '大四' | '硕士' | '博士';
  campus: '岳麓山' | '潇湘' | '南校区' | '天心' | '湘雅';
  college: string;             // 学院名，下拉选择
  height: number;              // 身高 cm
  heightPrefMin: number;       // 可接受最低身高
  heightPrefMax: number;       // 可接受最高身高
  hometown: string;            // 家乡省份
  crossCampus: boolean;        // 是否接受跨校区
  sameCollege: '接受' | '不接受' | '无所谓';

  // 模块二~六：1-7 Likert 量表题
  // 每题结构: { self: number, partner?: number, important: boolean }
  // self = 自己的特征/态度, partner = 对伴侣的期望（部分题有）
  // important = 是否标记为重要（默认 false）
  likert: Record<string, {
    self: number;
    partner?: number;
    important: boolean;
  }>;

  // 模块六：兴趣爱好（多选，最多5项）
  interests: string[];
  interestOverlap: { self: number; important: boolean }; // 希望爱好重合度

  // 模块七：核心特质（多选，最多5项）
  selfTraits: string[];
  partnerTraits: string[];
  partnerTraitsImportant: boolean;

  // 元数据
  school: 'csu';              // 预留跨校扩展
  seasonal?: string;           // 季节趣味题（选填）
  shootYourShot?: string;      // 心动邮箱（选填功能）
}
```

## 四、UI 组件规范

### 4.1 「☆ 设为重要」按钮

每道 Likert 题下方都有这个按钮。用户可以把对自己最重要的题标记出来（不限数量），匹配算法会给这些题的维度权重翻倍。

```html
<button class="importance-btn" onclick="toggleImportant('hustle', this)">
  <span class="star">☆</span> 设为重要
</button>
```

```css
.importance-btn {
  background: none; border: none; cursor: pointer;
  font-size: 12px; color: #777683; padding: 6px 0;
  transition: color .2s;
}
.importance-btn.on { color: #a43c12; }
.importance-btn.on .star { content: '★'; }
```

激活后文字变为「★ 已标为重要（匹配加权）」，颜色变为 secondary。

### 4.2 双轨题（自己 + 伴侣偏好）

部分题同时问"你自己"和"你希望伴侣"，用两行滑块实现：

```html
<div class="dual-slider">
  <div class="slider-row">
    <span class="slider-label-prefix">我自己：</span>
    <input type="range" min="1" max="7" value="4" />
  </div>
  <div class="slider-row" style="margin-top: 8px;">
    <span class="slider-label-prefix">希望伴侣：</span>
    <input type="range" min="1" max="7" value="4" />
  </div>
</div>
```

两行滑块共享同一个左端/右端标签，节省空间。「我自己」滑块用主色，「希望伴侣」滑块用浅一号的色调做区分。

### 4.3 滑块初始状态

沿用之前的设计：滑块 thumb 初始为半透明（`untouched` 状态），用户首次拖动后才变实色并计入已答题数。

### 4.4 进度条


## 五、数据存储与匹配算法对接

### 5.1 题目分类（给算法用）

| 类别 | 用途 | 包含题目 |
|------|------|---------|
| **硬过滤** | 匹配前过滤，不匹配的直接排除 | 性别、性取向、跨校区、同学院、身高范围、饮食要求 |
| **相似度维度** | 余弦相似度计算 | 所有 Likert 题的 self 值 |
| **偏好匹配** | 用 partner 值和对方 self 值的距离 | 双轨题的 partner 部分 |
| **加权乘数** | important=true 的题权重 ×2 | 用户自行标记 |
| **展示用** | 不参与算法，用于匹配报告 | 兴趣爱好、季节题、核心特质 |

### 5.2 匹配原因生成规则

匹配报告中展示 3 条原因，优先级：
1. 共同标记为重要的题 + 答案接近 → "你们都很看重 XX，而且态度高度一致"
